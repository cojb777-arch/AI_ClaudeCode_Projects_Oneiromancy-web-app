# 🌙 夢見録 — 夢日記 & 夢占い Webアプリ

夢を記録し、夢占い（夢判断）で読み解くためのWebアプリです。
ビルド不要の静的サイト（HTML / CSS / JavaScript のみ）で、データはすべてお使いのブラウザ内（localStorage）に保存されます。

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| ① | 内蔵の夢占いデータ | 定番の夢シンボル約120語（蛇・歯が抜ける・空を飛ぶ…）の解釈を内蔵 |
| ② | 夢占い本のアップロード | `.txt` `.md` `.csv` `.json` `.pdf` の夢占い資料を複数登録でき、診断時に参照 |
| ③ | 夢の診断 | キーボード記入または**音声入力**した夢を、辞書照合＋**Claude AIによる夢判断**で診断 |
| ④ | 日付つき保存 | 日付を指定してブラウザ内に夢日記として保存 |
| ⑤ | ダウンロード / アップロード | 夢日記データをJSONでいつでもバックアップ・復元・端末間移行 |
| ⑥ | Google Drive OCR | Driveの所定フォルダに入れた手書き夢日記の写真を一括OCR |
| ⑦ | OCR→診断→自動記録 | OCRしたテキストを夢診断し、日付を推定して日記に自動登録 |

## 使い方（起動）

静的ファイルなので、Webサーバーで配信するだけで動きます。

```bash
# 方法1: Python
python3 -m http.server 8000
# → http://localhost:8000 を開く

# 方法2: Node.js
npx serve .
```

> ⚠️ 音声入力・Google Drive連携は、ブラウザの仕様上 **https または http://localhost** でのみ動作します。`file://` で直接開いた場合は一部機能が使えません。
>
> 💡 GitHub Pages（Settings → Pages → Branch を選択）で公開すると、スマホからも使えて便利です。

対応ブラウザ: Chrome / Edge / Safari（音声入力はWeb Speech API対応ブラウザが必要）

## 各機能の設定

### ✨ AI夢判断・高精度OCR（Claude API）

1. [Claude Platform](https://platform.claude.com/) でAPIキーを取得
2. アプリの「⚙️ 設定」タブでAPIキーを入力して保存

APIキーはブラウザのlocalStorageにのみ保存され、Anthropic API以外には送信されません。
共有PCでは使用後に設定から削除することをおすすめします。

APIキーがない場合も、辞書照合による診断と Tesseract.js による無料OCRは利用できます。

### 🎤 音声入力

設定不要です。「🎤 音声入力を開始」を押してマイクの使用を許可し、夢の内容を話してください（日本語認識）。

### 📕 夢占い本のアップロード

「📚 夢占い辞書」タブからファイルをアップロードします。以下の形式を自動解析します。

- **1行形式**: `蛇：生命力と金運の象徴。…` / `【蛇】生命力と金運の象徴。…`
- **CSV**: `キーワード,意味` の2列
- **JSON**: `[{"kw": "蛇", "meaning": "…"}, …]` または `{"蛇": "…", …}`
- **見出し形式**: `# 蛇` や `■蛇` の見出し行 ＋ 続く本文の段落
- **PDF**: テキストを抽出して上記の形式で解析（画像スキャンのPDFは不可）

登録した本の内容は、辞書照合とAI夢判断の両方で参照されます。

### ☁️ Google Drive 連携（手書き日記のOCR取り込み）

手書きの夢日記をスマホで撮影し、Google Driveの所定フォルダ（既定: `夢日記`）に入れておくと、
「📖 夢日記一覧」→「☁️ Google Driveから手書き日記を取り込む」で一括処理できます。

処理の流れ: フォルダ内の画像を取得 → OCR（Claude Vision または Tesseract.js）→ 夢診断 → 日付を推定して日記に自動登録。
一度取り込んだ画像は記録され、次回はスキップされます。

**OAuthクライアントIDの取得手順:**

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス」→「ライブラリ」で **Google Drive API** を有効化
3. 「APIとサービス」→「OAuth同意画面」を設定（テストユーザーに自分のGoogleアカウントを追加）
4. 「認証情報」→「認証情報を作成」→「OAuthクライアントID」→ 種類は **ウェブアプリケーション**
5. 「承認済みのJavaScript生成元」にアプリのURL（例: `http://localhost:8000` や GitHub PagesのURL）を追加
6. 発行された `xxxx.apps.googleusercontent.com` をアプリの「⚙️ 設定」タブに入力

要求スコープは読み取り専用（`drive.readonly`）のみで、Driveへの書き込みは行いません。

## データの保存場所と注意

- 夢日記・アップロードした本・設定はすべて **ブラウザのlocalStorage** に保存されます（サーバー送信なし）
- ブラウザのデータ消去で失われるため、**定期的に「⬇️ 日記をダウンロード」でバックアップ**してください
- localStorageの容量は約5MBです。大きな本を多数登録する場合はご注意ください

## ファイル構成

```
index.html        … アプリ本体（画面）
css/style.css     … スタイル
js/dictionary.js  … 内蔵の夢占い辞書データ（追加・編集可能）
js/app.js         … アプリのロジック
```

内蔵辞書は `js/dictionary.js` の配列に `{ kw, syn, fortune, meaning }` を追加するだけで拡張できます。
